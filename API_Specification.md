# ТЕХНИЧЕСКАЯ СПЕЦИФИКАЦИЯ API — СЕТЬ «КОНТИНЕНТ»

## 1. МОДУЛЬ ОНЛАЙН-РЕЗЕРВИРОВАНИЯ (CLICK & COLLECT)

### 1.1. Метод POST /api/v1/orders/hold
Инициирует временное (15 минут) холдирование дефицитного товара на складе конкретной аптеки до момента успешной онлайн-предоплаты.

* **Тип запроса**: Спецификация REST API.
* **Заголовки (Headers):**
  * `Content-Type: application/json`
  * `Authorization: Bearer <token>` (Обязательный JWT-токен авторизованного клиента).
 
## 2. МАТРИЦА КЛИЕНТСКИХ ОШИБОК (RESPONSE ERRORS 4XX)

### 2.1. Ошибка 400 Bad Request (Нарушение валидации контракта)
* **Описание**: Фронтенд прислал запрос, в котором один из обязательных параметров передан в неверном типе данных или нарушен синтаксис JSON (например, вместо UUID передан лидирующий нуль).
* **Response Body (JSON):**
```json
{
  "code": 1001,
  "message": "store_id: expected valid UUID format"
}
```
### 2.2. Ошибка 422 Unprocessable Entity (Бизнес-ошибка выполнения)
* **Описание**: Запрос синтаксически корректен, но бизнес-логика не может быть исполнена из-за отсутствия физического остатка товара на складе выбранной аптеки.
* **Response Body (JSON):**
```json
{
  "code": 3001,
  "message": "Товар не может быть холдирован: отсутствует в наличии и недоступен для заказа"
}
```
### 2.3. Ошибка 409 Conflict (Конфликт параллельных запросов)
* **Описание**: Повторный дублирующий запрос к заказу, который уже заблокирован системой и находится в статусе холдирования (`HOLDING`). Защита от двойного клика кассира или пользователя в приложении.
* **Response Body (JSON):**
```json
{
  "code": 4001,
  "message": "Запрос дублирует уже заблокированный заказ"
}
```

### 2.4. Ошибка 429 Too Many Requests (Превышение лимита частоты запросов)
* **Описание**: Сработал алгоритм ограничения частоты запросов (Rate Limiter). С текущего аккаунта или IP-адреса зафиксирована аномальная активность (более 5 запросов в секунду на метод холдирования). Запрос заблокирован на уровне шлюза.
* **Response Body (JSON):**
```json
{
  "code": 5001,
  "message": "The request limit has been reached, please wait."
}
```

### Матрица серверных ошибок (response errors 5хх)
## Ошибка 500 Internal Server Error (Критический сбой сервера)
* **Описание**: Критический сбой на уровне инфраструктуры бэкенда сети «Континент». Запрос корректен, но выполнение заблокировано на уровне железа или внутренних микросервисов.
* **Response Body (JSON):**
```json
{
  "code": 9001,
  "message": "An internal server failure occurred"
}
```





























* **Матрица клиентских ошибок (Response Errors 4xx):**

  * **Ошибка 400 Bad Request (Неверный формат параметра)**
    * *Описание*: Параметр `order_id` передан в нетекстовом или невалидном формате (ожидалось целое число `INT`).
    * *BODY:*
      ```json
      {
        "code": 1001,
        "message": "order_id: ожидалось целое число (INT)"
      }
      ```

  * **Ошибка 401 Unauthorized (Отсутствие авторизации)**
    * *Описание*: В заголовках запроса (Headers) не передан токен авторизации или он протух.
    * *BODY:*
      ```json
      {
        "code": 2001,
        "message": "Пользователь не авторизован в системе Континент"
      }
      ```

  * **Ошибка 404 Not Found (Заказ не существует)**
    * *Описание*: Запрос прошел авторизацию, но указанного `order_id` нет в базе данных аптеки.
    * *BODY:*
      ```json
      {
        "code": 4004,
        "message": "Заказ с указанным идентификатором не найден"
      }
      ```
